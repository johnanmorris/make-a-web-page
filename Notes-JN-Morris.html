Lesson 1: The Basics of the Web

The Web

The web is essentially a collection of documents. Most of these documents are HTML documents (but there are also PDFs, image files, etc). A user uses a web browser on his or her computer to view the Internet. In order to do this, the user’s computer must send a request with HTTP (hypertext transfer protocol) to servers which store the HTML pages. The Internet is just a collection of many servers connected together, interpreted through a web browser.

HTML stands for HyperText Markup Language. An HTML document has three basic components:

  • text, or the main content of a page;
  • markup, the code telling the computer what a page should look like; and
  • links to other pages.

HTML Markup

The basic unit of HTML markup is the tag. A tag is always surrounded by angle brackets (< >). Most tags have an opening tag and a closing tag. A closing tag has a forward slash after the first bracket (</ >). Everything between tags is known as an element.

Some tags have what is known as an attribute. An attribute is a property of an HTML element. For example, the anchor tag, used to create links (like this one to Google), has an attribute known as href. So, a link would look like this:

<a href="url">link text</a>

Tags can have more than one attribute; for instance, the <img> tag, used for linking to and displaying images, usually has two attributes, src (for source) and alt (for alternate). The src tag is used to link to the url of the image, while alt displays text if a link to the image is broken or a page is being viewed by a blind person.

The <img> tag is a type known as a void tag. This simply means that there is no content (text) within the tag. Void tags do not need closing tags (and can be “self-closed” with a forward slash immediately before the final angle bracket). Another void tag is used for line breaks, <br/>.

Inline Tags vs. Block Tags

Inline tags are tags which occur within a text. Many common tags are inline tags, such as:

  • line break: <br/>
  • image: <img>
  • italics: <em>
  • bold: <b> or <strong>.

On the other hand, a block tag, is one that sections off an entire section of text, creating an invisible “box” around an element. The most common block tag is probably the paragraph, or <p>.

Span and Div

The difference between inline and block tags can be illustrated by comparing the <span> and <div> tags. Both tags are container tags—they surround an element in a certain style. However, <span> is an inline tag, existing within a line or block, while <div> is a block tag, creating a “box” or block around an element.

Anatomy of a Page

There are many tags, but all HTML pages have the following parts which make up the skeleton or backbone of a page:

  • <!DOCTYPE HTML> : tells you what the document type is
  • <html> : starts and ends each HTML document
  • <head> : This tag contains the metadata of a page. This can include the title (enclosed by <title> tags), links to any referenced CSS or JavaScript pages, and the like.
  • <body> : These tags enclose the main content of the page—all the text, images, and links within a page.

Lesson 2: Creating Structured Documents with HTML

Languages of the Web

When you write a webpage, you are using two basic languages, HTML and CSS:

  • HTML provides the structure of a page—what the headings, paragraphs, links, and images are.
  • CSS, or Cascading Style Sheets, tells the browser how a page is styled: the font size and color of each element, the position of elements, borders, and so on.

The Structure of an HTML Page

An HTML page can be conceptualized with a tree model, with elements branching out within or beneath other elements. For example, the paragraph you are reading right now is nested as follows:

body → div → div → p

This is a simplified example, leaving out many parallel branches, but illustrates this nesting structure. You can view the structure of any webpage by using the “Inspect Element” tool in most browsers.

Elements as Boxes

The “Inspect Element” tool will also show you another feature of elements: they are all boxes. This is actually a useful feature of HTML and CSS, as it allows you to easily rearrange elements on a page. It also enables you to break down the design of a a page into blocks, which has smaller blocks nested inside of it.

Treating elements of a page as a gives you a convenient starting point for figuring out how to code a design. For example, this can help you find the best place for each div.

Lesson 3: Adding CSS Style and HTML Structure

CSS

Along with HTML, CSS is one of the most common languages used in web development. CSS stands for Cascading Style Sheets. It’s cascading because the styling rule of a parent element is applied to all its children (Source). This is also known as inheritance. If there is a more specific rule, though, then the styling of the parent element does not get passed down. For instance, if I wanted to make all text in a paragraph (i.e., in the <p> tags) blue, all of the text would also be blue, whether it was bolded, italicized, or underlined. However, if I wrote a rule in CSS that made all underlined text the color red, then that rule would override the paragraph styling, because it’s more specific.

The syntax of CSS is a bit different than HTMl, though some of the code is similar. It's basically composed of three parts:

  • selector: The element of the page a particular style will apply to. This can be HTML (e.g., p, em, ul), or a user-defined class.
  • property: This tells the browser what particular aspect of the selector is being styles (e.g., text size, color, etc.).
  • value: The value of a given property.

The basic template for CSS, then, would look like this:

h2 {
font-family: Georgia, serif;
}

If I wanted to style a class, though, it would look more like this:

.class {property: value;}

Note: If you write CSS all in one line like this, the computer can still read it. However, in other to make it easier for humans to read, it’s recommended to indent, by placing each property and value on its own line.

Use CSS to Avoid Repetition

When doing any coding, one of the primary goals is to prevent repetition. Using selectors and classes in CSS allows you to avoid repetition.

Say you wanted to emphasize certain paragraphs by changing the font face, size, and color. If I were just working with HTML, I would have to put something like this in front of each paragraph, each time you changed these values:

<font face="Helvetica" color="red" size="15">

Then, to get the style of the normal paragraphs, I would have to put in the <font> HTML again, by writing something like this:

<font face="Georgia" color="black" size="10">

If I had to change a lot of paragraphs this way, it would get very tedious, I’d have to write a lot more code, and I might be prone to making more mistakes.

With CSS, though, I can create a class in a single stylesheet, and use that stylesheet over several webpages (maybe even an entire website) In this case, I could do something like this:

.bigred {
font-family: Helvetica, sans-serif;
size: 20px;
color: red;
}

Then, any time I wanted to use this style, I can give the paragraph the class "bigred":

<p class="bigred">This is an emphasized paragraph.</p>

Using classes in CSS also makes it easy to make changes to a design—so for example, if I decided I wanted to indent the "bigred" class, I could just change the class once in the CSS file, rather than trying to find each instance in the HTML.

CSS and the Box Model

As stated previously, everything on the web is a box. Well, a page element is not just a simple box—it’s a bit more complicated than that! For any given element, a total of three other boxes surround it. Starting from the inside and working out, an element is composed of each of the following:

  • Content: The text or image
  • Padding: Acts as a buffer around the content
  • Border: Surrounds the content and image
  • Margin: Clears an area around the border

Each of these elements can be controlled independently of the other in HTML. However, that gets complicated quickly. To simplify matters, the following CSS can be used:

*{box-sizing: border-box;}

With this code, the overall width of a box is not affected by changes to the padding and border. This code does not take the margin into account. To make this work on older browsers, additional code is needed.

Another useful bit of CSS makes divs appear next to each other, rather than take up the entire width of the page. To do this, place the following declaration in the relevant selectors:

display: flex;

Process

Here are the basic steps of the web design process:

  1. Look for natural boxes.
  2. Find styles and semantic elements that can be repeated.
  3. Write HTML.
  4. Write CSS, and apply styles starting at the biggest elements and working down the the smallest ones.
  5. Fix things as you go along by continuously looking at how your changes affect the design.
